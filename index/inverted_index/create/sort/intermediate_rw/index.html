Intermediate codec for external sorting.

This module provides serialization and deserialization logic for handling intermediate data during the sorting process. The serialization format is as follows:

[magic][bitmap leading zeros][item][item]...[item]
   [4]          [4]              [?]

Each [item] is structured as:
[value len][value][bitmap len][bitmap]
    [8]       [?]       [8]        [?]

The format starts with a 4-byte magic identifier, followed by a 4-byte bitmap leading zeros count, indicating how many leading zeros are in the fixed-size region of the bitmap. Following that, each item represents a value and its associated bitmap, serialized with their lengths for easier deserialization.
